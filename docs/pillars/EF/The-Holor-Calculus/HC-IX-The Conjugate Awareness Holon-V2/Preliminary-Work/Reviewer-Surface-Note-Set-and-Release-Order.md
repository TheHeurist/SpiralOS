# Reviewer Surface Note Set and Release Order

## Core utterance

A public release must not begin as an archive dump.

It must begin as a lawful invitation:
finite enough to be navigable,
exact enough to be challenged,
and alive enough to preserve what the work is for.

Thus publication proceeds in surfaces and phases, not in undifferentiated bulk.

## The three surfaces

### Reviewer surface

This is the first point of contact for hostile, cautious, reductionist, or strictly formal readers.

Its job is not to contain everything.

Its job is to let such a reader answer, quickly:

* What is being claimed?
* What is not being claimed?
* What is theorem, proof sketch, design axiom, or open hypothesis?
* What definitions and dependencies matter most?
* Where does the work remain frontier?

A healthy first reviewer surface should include:

* précis
* reviewer guide
* publication surface and claim hierarchy
* core formal bridge notes
* explicit non-claims
* provenance / attestation link

### Bridge surface

This surface translates without flattening.

It is where a serious reader can cross from compact review into the deeper architecture.

It should include:

* fascia
* peering
* shared conjugation
* worked examples
* relation maps
* diagram legends
* explanatory bridge prose

Its job is not to prove everything.
Its job is to prevent premature misunderstanding.

### Field surface

This surface preserves the living intelligibility of the work.

It should include:

* asymptotic affirmation
* escrow note
* charter materials
* invocation materials

Its job is not to satisfy flatland first.
Its job is to preserve what the work is for.

## Release order

### Phase 1 — Internal stabilization

Before public release:

* finalize canonical markdown blocks
* align filenames, statuses, and dependencies
* verify claim-status labels
* cross-link the spine

### Phase 2 — Reviewer surface

Release first:

* précis
* reviewer guide
* claim hierarchy
* core formal bridge notes

This is the lawful first invitation to scrutiny.

### Phase 3 — Bridge surface

Release next:

* fascia
* peering
* shared conjugation
* worked examples
* diagrams and relation maps

This permits serious movement into the architecture.

### Phase 4 — Field surface

Release then:

* asymptotic affirmation
* escrow note
* charter / invocation companions

This keeps the field alive in public rather than hidden for respectability.

### Phase 5 — Expansion

Only after the above:

* promote theorem-grade materials
* widen glossary and supplements
* issue companion bundles
* answer objections using exact claim-status rather than defensive flattening

## Platform roles

### GitHub

GitHub should serve as:

* living repository
* navigable braid of notes
* visible provenance surface
* iterative refinement space

It should emphasize:

* structure
* cross-links
* witness of development
* readable bridge material

### Zenodo

Zenodo should serve as:

* stable citation surface
* compact reviewer-facing deposit
* versioned hardened release

It should emphasize:

* précis
* reviewer guide
* attestation
* compact integrity
* stable versioning

### Companion surfaces

Other publication surfaces may carry:

* broader narrative continuity
* ceremonial material
* field-intelligible introductions
* public-facing synthesis

They should remain faithful without carrying the entire formal burden.

## Reviewer-safe rule

When challenged, answer with exact surface and exact burden.

Do not say:
“it is all one thing.”

Say:

* this is reviewer surface
* this is bridge surface
* this is field surface

And:

* this is theorem
* this is proof sketch
* this is design axiom
* this is open hypothesis
* this is charter
* this is invocation
* this is trace

This is how the work passes through hostile review without interior amputation.

## Boundary of claim

This note does **not** claim that all parts are ready for final release now.

It does **not** claim that every reader should meet the same surface first.

It does **not** subordinate the field surface to reviewer permission.

It does claim that lawful release now requires staged surfaces, explicit burdens, and visible provenance.
